# TrackMate — Documentation Index

TrackMate is bundled with Fiji (core detectors and trackers). Cellpose and StarDist integration require additional update sites. All scripting uses the TrackMate Java API directly — not `IJ.run()`.

## Critical pitfalls

- **`Image must be 2D over time, got an image with multiple Z.`** — the input
  was loaded with planes in Z instead of T (typical when an image sequence
  was imported via `File › Import › Image Sequence`). TrackMate-Cellpose,
  TrackMate-StarDist, and any 2D+T detector reject this. Fix with
  `IJ.run(imp, "Properties...", "channels=1 slices=1 frames=N ...")` BEFORE
  building `Settings`. Full pattern with Bio-Formats prevention:
  [`/app/skills/imagej_groovy_patterns/pitfall_z_vs_t_dimensions.md`](../imagej_groovy_patterns/pitfall_z_vs_t_dimensions.md).

- **`The tracker factory is null.`** / **`unable to resolve class SparseLAPTrackerFactory`** /
  **`unable to resolve class ManualTrackerFactory`** — TrackMate requires
  `settings.trackerFactory` even for detection-only pipelines, AND the tracker
  package was renamed (`tracking.sparselap.*` → `tracking.jaqaman.*`) in v8+.
  Use the reflection-based loader pattern in `CELLPOSE_DETECTOR_API.md` § Pitfall C5
  rather than hardcoded imports — it tries multiple class names at runtime and
  survives future renames.

- **`Could not locate Cellpose mask image generated by TrackMate-Cellpose.`** —
  TrackMate-Cellpose does NOT open the segmentation as a window; it writes the
  label image to `/tmp/TrackMate-cellpose_<random>/<basename>_cp_masks.{tif,png}`.
  Read that file directly. See `CELLPOSE_DETECTOR_API.md` § Pitfall C7.

- **`RoiManager.addRoi() expected an ij.gui.Roi but received fiji.plugin.trackmate.SpotRoi`** /
  **`ROIs extracted ... 0`** — `Spot.getRoi()` returns a `SpotRoi`, not an
  `ij.gui.Roi`, and Cellpose detections may yield zero usable contours anyway.
  Skip ROI extraction entirely for Cellpose; treat the mask file from C7 as the
  ground-truth output. See `CELLPOSE_DETECTOR_API.md` § Pitfall C8.

- **`unable to resolve class inra.ijpb.measure.region2d.IntensityMeasures`** —
  some MorphoLibJ classes don't compile in every Fiji build even when MorphoLibJ
  is installed. Use plain pixel iteration for label-image measurements
  (no MorphoLibJ import required). See `CELLPOSE_DETECTOR_API.md` § Pitfall C9.

For the canonical end-to-end pattern that addresses all of the above in one
script, copy [`GROOVY_WORKFLOW_CELLPOSE_SINGLE_IMAGE.groovy`](GROOVY_WORKFLOW_CELLPOSE_SINGLE_IMAGE.groovy).
