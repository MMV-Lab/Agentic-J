# Data, Chat History & Reports

## Where your files live

Everything the agent writes inside the container appears immediately on your computer under `./data/`. No manual copy is needed. The project folder structure is created automatically by the agent when it starts a new analysis (`setup_analysis_workspace`).  You can also place your source images directly in `./data/` or any subdirectory and tell the agent the path.

```
./data/
├── projects/                   ← one subfolder per analysis project
│   └── my_project/
│       ├── raw_images/         ← copy of your source images
│       ├── processed_images/
│       │   ├── channels/       ← channel-split or pre-processed TIFFs
│       │   └── montages/       ← overview images
│       ├── scripts/
│       │   ├── imagej/         ← generated Groovy scripts
│       │   └── python/         ← generated Python (stats, plots) scripts
│       ├── data/               ← measurement CSVs and results tables
│       ├── figures/            ← plots and publication figures
│       ├── logs/               ← execution logs
│       ├── state_ledger.json   ← persistent project memory (parameters, paths, notes)
│       └── QA_Checklist_Report.md  ← generated by QA reporter (if enabled)
├── chats/                      ← chat history database
│   ├── checkpoints.db          ← full conversation history (SQLite)
│   └── index.json              ← session index (titles, timestamps)
└── imagentj_debug.log          ← application debug log
```

---

## Chat history

Conversations are stored persistently and survive container restarts. The left sidebar in the chat UI lists all previous sessions. Click any session to load it. This helps to keep the context of any specific topic in place.

---

## Usage reports

The **"Save Usage Report"** button (bottom of the chat panel) saves a JSON file containing token counts, cost estimates, and model usage for the selected conversation. Use this to track API spending.

The file is saved to a location of your choice on the host.

---

## Reporting issues

The **"Report Issue"** button opens a dialog where you can:
1. Describe what went wrong.
2. Select which conversations to include (exported as JSON or ZIP).
3. Click **"Send Report"** — the report is emailed to `agentj.help@gmail.com`.

 If email fails (e.g. your network blocks outbound SMTP), the dialog will offer to save the report to a file instead — you can then email it manually to `agentj.help@gmail.com`.

The report contains: your description, conversation transcripts, script code, and system info. **No image data or raw file contents are included.**
